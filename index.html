The game is afoot

Let's use IBM Watson's Dialog API to gather the facts of the case. We'll apply these facts to IBM Watson's Concept Insights API to generate clues. All of the facts and clues can be stored together with the Hackpad API.

// Example

The potential of the Concept Insights API has been proven with some manual queries. Objects such as a coin, key, letter, ring, pocket watch, and towel can be related to single concepts in Wikipedia. Many of these objects can have multiple meanings, but the physical objects were used in this example. What concept connects all of these objects? a Chain

  • Coins can be ornaments on a necklace chain.
  • Keys are carried on a keychain.
  • Letters can be chain letters.
  • Rings are links of a chain.
  • Pocket watches are attached to a chain.
  • Towels are chains of thread.

We can agree that "chain" would not be useful as a clue, but it is an obscure and complex relationship. Combining objects, people, and places should prove more useful for clue generation. Luckily, there is tons of Sherlock Holmes data in Wikipedia!
